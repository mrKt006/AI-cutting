# 公网商业版架构边界

## 当前安全状态

当前版本是本地单人应用，不是可直接开放公网的 SaaS。默认中间件只接受本机 Host 和本机客户端地址。即使误用 `--host 0.0.0.0`，远程请求也会被拒绝。

`AI_CUTTING_ALLOW_UNSAFE_REMOTE=1` 只能用于受信任内网临时调试，不代表应用具备公网安全性。

## 公网版必须替换的本地实现

| 本地实现 | 公网版目标 |
| --- | --- |
| `jobs/<job_id>/job.json` | PostgreSQL 任务、状态、归属、用量和计费表 |
| Web 进程内线程 | Redis/数据库持久化队列与独立 Worker |
| 本地 `jobs/` 媒体 | 对象存储、租户前缀、生命周期和服务端加密 |
| `settings.local.json` 明文密钥 | 云 KMS/Secrets Manager 信封加密 |
| 可猜测任务 URL | 登录会话、资源归属校验和不可枚举外部 ID |
| 本地文件下载 | 短时签名 URL、权限校验和下载审计 |
| 单机信号量 | 套餐配额、租户并发、全局 Worker 容量控制 |

## 核心数据模型

- `users`：账号状态、认证提供方和安全事件时间。
- `organizations`、`memberships`：团队和角色。
- `projects`：预设、词库和默认处理策略。
- `jobs`、`job_items`：租户归属、阶段、幂等键、错误类型和取消状态。
- `artifacts`：对象存储 key、类型、字节数、校验和、保留期限和删除状态。
- `encrypted_credentials`：服务商、密文、KMS key 版本和最后使用时间。
- `usage_events`：ASR 秒数、LLM token、Worker 秒数、存储和下载流量。
- `feedback_events`：明确授权状态、数据版本、撤回和删除审计。

所有读取和修改必须同时使用资源 ID 与 `tenant_id` 查询。只验证“这个 ID 存在”不构成权限校验。

## 任务与幂等

1. Web 完成上传校验后创建任务和不可变参数快照。
2. 每个阶段使用 `job_item + stage + input_hash + algorithm_version` 作为幂等键。
3. Worker 领取带租约的任务；租约超时后其他 Worker 才能接管。
4. 火山和 LLM 调用前检查本地/数据库结果，调用后先持久化响应再推进状态。
5. 产物先写临时对象，校验时长、大小和 checksum 后原子发布。
6. 计费事件使用唯一幂等 ID，重试不能重复记账。

## 密钥与隐私

- 浏览器永远不接收服务端 ASR/LLM Key。
- 数据库只保存密文和 KMS 元数据，不保存可直接使用的明文。
- Worker 按任务临时解密，变量生命周期结束后立即释放。
- 日志、错误追踪、分析平台和训练导出统一执行敏感字段过滤。
- 用户删除任务后立即撤销下载，随后异步删除媒体、检查点和派生数据。
- 训练数据必须使用单独授权，撤回后从待训练集和后续训练版本中排除。

## 上线门槛

1. 20 条固定视频质量集通过，并有人工复核证据。
2. 资源越权、任务 ID 枚举、上传恶意文件和签名 URL 泄露测试通过。
3. Worker 崩溃、队列重复投递、ASR/LLM 超时和对象存储失败恢复通过。
4. 单任务成本上限、套餐配额和异常用量告警通过。
5. 隐私说明、数据删除、训练授权和客服处理流程上线。

在这些条件满足前，不应移除本机访问安全门或对公网开放端口。
