# 本地专业版质量评估

## 生成报告

```powershell
python scripts/build_evaluation_report.py
```

报告默认写入 `evaluation/report.local.json`。该文件包含本地任务 ID、源文件指纹和用户修改统计，因此不会进入 Git。

也可以打开 `http://127.0.0.1:8000/evaluation` 查看同一套验收进度，并直接进入每条视频的精修台。完整看完并确认结果后，点击精修台顶部的“完成复核”。之后若再次修改，该复核状态会自动失效，必须重新确认。

## 固定测试集门槛

通过本地专业版质量验收前，至少需要：

1. 20 条来源指纹不同、成功生成成片的代表性口播视频。
2. 20 条使用当前 AI 决策格式生成的 `ai_decisions.json`。
3. 20 条在精修台明确点击“完成复核”，且确认后未再次修改的 `training_feedback.json`。
4. 视频覆盖短句、长句、中英文、数字、专有名词、卡壳、半句重说、完全重复、语义重复和不同停顿速度。

旧任务可以验证兼容性，但没有新版决策记录和人工反馈时，不能用于证明 AI 准确率。

## 核心指标

- 错字纠正量及人工再次修改量。
- AI 自动删除量、用户恢复量和删除恢复率。
- 用户新增删除量，用于估算漏删。
- 用户拆句、合句和时序修改量，用于估算断句返工。
- 内容标题与封面标题修改量。
- prompt、completion 和总 token 用量。
- 每分钟视频处理时间与人工精修时间。后两项需要后续任务记录精确阶段耗时后才能正式统计。

删除恢复率只能作为错误删除的近似指标。用户没有进入精修台，不代表 AI 删除正确；因此必须由测试人员完整观看并保存一次复核结果。
