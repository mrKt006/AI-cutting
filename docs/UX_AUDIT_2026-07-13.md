## 结论

当前版本已经具备完整 MVP 的形状，但还不适合直接作为“无需指导即可使用”的公网产品。主要风险不是页面是否高级，而是运行环境不可用时缺少恢复路径、编辑状态可能互相矛盾、任务与导出不可管理、精确预览会静默降级，以及公网模式没有用户和文件隔离。

## P0：当前交付阻塞

### P0-1 FFmpeg 链接损坏导致核心流程不可运行

- **证据**：首页与设置页显示“FFmpeg 未检测到”，开始处理按钮禁用；封面预览失败；时间线连续代理无法重新生成。
- **根因证据**：WinGet 的 `ffmpeg.exe` 符号链接仍存在，但目标目录不存在；Python `shutil.which("ffmpeg")` 返回空。
- **影响**：新任务、精确预览、封面渲染、连续时间线预览都不可用。
- **验收**：应用启动时能检查 `ffmpeg -version` 和 `ffprobe -version`，不仅检查路径；提供明确目标路径、诊断和修复入口；核心自检全部通过。

### P0-2 公网部署没有身份、任务和密钥隔离

- **证据**：当前没有登录鉴权；设置、任务、下载和精修 URL 均按本地单人模式开放。
- **影响**：公网用户可能读取或修改其他人的任务；服务端共享的火山/LLM Key 会被所有访问者共同使用。
- **验收**：上线前增加用户身份、任务所有权校验、密钥归属与加密、下载鉴权、速率限制和审计记录。

### P0-3 上传后端缺少文件类型和体积上限

- **证据**：前端有 `accept=video/*`，但创建任务的后端只检查文件非空，没有强制 MIME、扩展名、文件头、单文件和总大小限制。
- **影响**：公网可被超大文件耗尽磁盘，也可能接收非视频文件。
- **验收**：流式上传限制、格式白名单、文件头校验、剩余磁盘检查、任务配额和可读错误提示。

## P1：高优先级可靠性问题

### P1-1 精修台三处文字状态会互相矛盾

- **复现 A**：右侧文本框修改字幕后按 `Ctrl+Z`，左栏和预览恢复，但右侧文本框仍保留撤销前文字，页面却显示“已保存”。
- **复现 B**：双击画面字幕编辑，按一次退格，文字未按预期删除且光标回到开头。
- **复现 C**：左栏进入编辑时，右侧检查器不会立即同步当前草稿。
- **影响**：用户无法判断最终保存的是哪一份文字，继续输入可能覆盖刚撤销的结果。
- **验收**：字幕文本只有一个状态源；左栏、画面和检查器全为该状态的受控视图；撤销、重做、保存后逐项一致。

### P1-2 自动保存、保存按钮与内存撤销边界不清楚

- **证据**：修改约 1.4 秒后自动保存，同时顶部仍有“保存”；结构修改撤销后可能显示“有未保存修改”。撤销栈只存在当前页面内存中。
- **风险**：自动保存了误删后刷新页面，用户无法撤销；显式保存按钮的意义不清楚。
- **验收**：明确采用“自动保存 + 版本历史”或“手动保存 + 离开提醒”之一；若保留自动保存，至少提供最近版本恢复和持久化操作历史。

### P1-3 精确预览失败时静默显示 CSS 近似预览

- **证据**：样式页 `/api/style-presets/render-preview` 失败后直接隐藏精确图并显示 CSS 字幕，没有任何错误或“近似预览”标识。
- **影响**：用户以为看到的是最终 libass 效果，实际导出可能不同。
- **验收**：精确预览失败必须显示明确状态和重试；近似预览只能带清晰标识，不能伪装成最终效果。

### P1-4 预设删除无确认也无撤销

- **复现**：新建并复制预设后点击删除，预设立即消失并跳回默认，没有确认或撤销。
- **影响**：人工调试较久的预设可能误删。
- **验收**：确认对话框或软删除回收站；被任务引用的预设需提示影响。

### P1-5 失败任务没有恢复动作，任务可能永久停留在“处理中”

- **证据**：失败详情仅显示 traceback；没有重试、复制参数重新创建、取消、删除或诊断按钮。历史中存在长期 `running` 任务。
- **影响**：普通用户无法自救；批量任务中后续条目会一直“等待中”。
- **验收**：提供重试、取消、删除、从失败项继续、复制参数；应用启动时回收失联进程并把僵尸任务标成中断。

### P1-6 AI 断句结果可见明显语义破坏，旧任务缺少版本提示

- **真实任务证据**：存在“生意反而做起来不 / 是他们突然变勇敢是”“最高的打我们在广西”等边界错误。
- **影响**：核心自动剪辑质量不可直接交付；旧数据与新算法混在同一 UI，用户不知道是否应重新分析。
- **验收**：保证 token 只重分组、不丢字、不重字、顺序不变；显示分析版本与运行时间；旧任务提供“使用新算法重新分析”横幅和前后对比。

### P1-7 时间线长项目采用全量 DOM 重绘

- **证据**：`renderAll()` 同时重建分句、全文、时间线、检查器和输出；时间线结构变化会重建所有片段与监听器。
- **影响**：54 句尚可，数百句或长视频时拖动、裁切和选择会迟滞。
- **验收**：时间线与字幕列表虚拟化；拖动时只更新受影响片段；耗时代理生成放后台且不阻塞基本编辑。

### P1-8 批量标题只有第一条能看到预览

- **证据**：多视频会生成逐条标题输入，但首页预览逻辑只读取第一条标题。
- **影响**：第二条以后只能盲填，容易在导出后才发现换行和内容错误。
- **验收**：素材列表可选择当前视频，标题和封面预览随选择切换；每条显示是否已填写。

## P2：中优先级体验问题

### P2-1 “查看全部任务”并不是真正的全部

- **证据**：后端最近任务固定最多读取 8 条；首页展开只展示这 8 条，没有独立任务页、分页或搜索。
- **验收**：新增任务中心，支持分页、搜索、状态筛选、重试、取消和删除；首页仅保留最近 3 条。

### P2-2 任务日志直接暴露技术细节并存在运行时乱码

- **证据**：详情展示 Python 路径、项目绝对路径、完整命令和 traceback；成功日志中的中文目录被输出为连续的 Unicode 替换字符。
- **说明**：静态编码扫描通过，说明问题来自 Windows 子进程输出编码，不是源文件编码。
- **验收**：普通视图显示步骤和可行动错误；技术日志折叠；统一设置子进程 UTF-8；路径按相同编码解码；密钥继续脱敏。

### P2-3 运行中任务每 5 秒整页刷新

- **证据**：任务详情使用 meta refresh。
- **影响**：阅读日志时页面跳动、滚动位置和折叠状态丢失。
- **验收**：轮询状态 API 或 SSE，只局部更新状态和日志。

### P2-4 导出区累积展示全部历史导出，无法识别当前版本

- **证据**：精修台右侧同时列出多组同名 MP4、JPG、ASS 和 JSON。
- **验收**：当前版本置顶并分组；历史版本折叠；显示导出时间、版本和“设为当前/删除”。

### P2-5 内容标题与封面标题样式已独立，但初始文字相同仍容易误解

- **核对结果**：真实任务中内容标题和封面标题初始文字相同；样式、位置和预设是独立数据。
- **问题**：界面没有解释“默认复制内容，但之后独立编辑”，用户会认为两者仍然绑定。
- **验收**：首次进入时明确一次性来源；编辑后显示“独立修改”；提供“从内容标题同步一次”，不要持续双向绑定。

### P2-6 封面编辑依赖服务端 FFmpeg，失败后完全不可调

- **证据**：当前封面模式直接显示“封面预览失败”，用户无法继续可靠调整。
- **验收**：保留浏览器即时近似画布，后台精确预览作为校验；失败不应让整个封面编辑失去反馈。

### P2-7 设置页没有未保存离开保护

- **验收**：修改后离开页面给轻提示，或采用清晰可靠的自动保存。

### P2-8 FFmpeg 故障提示不可操作

- **证据**：页面只提示未检测到，没有显示检测路径、版本、修复按钮和重新检测。
- **验收**：设置页提供诊断明细、重新检测、一键打开安装说明；Windows 符号链接失效要单独说明。

### P2-9 移动端精修不是“轻量编辑”，而是桌面界面纵向堆叠

- **数据**：375px 下精修页高度约 2846px，58/82 个可交互控件小于 44px；预览、字幕和检查器无法同时看到。
- **验收**：移动端只保留播放、字幕改字、隐藏/恢复和导出；时间线与完整样式检查器改为底部面板或明确提示桌面使用。

### P2-10 样式预设移动端横向溢出且无法边调边看

- **数据**：375px 下页面高度约 2247px，主编辑器和表单存在横向溢出，27/68 控件小于 44px。
- **验收**：预览固定在可视区上部；参数用底部抽屉；消除横向滚动。

### P2-11 表单可访问名称不完整

- **证据**：样式页约 34 个可见控件、精修页约 16 个控件缺少关联 `label/for` 或 `aria-label`；视觉上的相邻文字不等于程序化标签。
- **验收**：每个输入、颜色、滑杆和搜索框都有唯一名称；错误信息与控件通过 `aria-describedby` 关联。

### P2-12 文本与时间线对比度不足

- **数据**：首页次级标题约 4.23:1；时间轴刻度约 3.23:1；棕色字幕片段上的浅色字约 4.2:1，均低于普通文本 4.5:1。
- **验收**：调整文字或背景色并运行自动对比度检查。

## P3：清理与一致性

- 编辑器工具按钮普遍约 30px 高，桌面尚可，触控目标偏小。
- “进入精修”在任务详情视觉上像禁用状态，需加强可点击性。
- “智能分析”“重新断句”的完成反馈短暂且位置不固定，需统一成持久任务状态或 toast。
- 首页设置文案写“重复内容只标记等待确认”，与当前“高置信度自动剪辑”的产品方向冲突，应明确自动模式与保守模式。
- 样式页精确预览每次输入后 650ms 请求一次服务端，连续拖滑杆时应取消旧请求并考虑鼠标松开后精确渲染。
- CSS 文件约 3245 行，存在大量硬编码颜色；建议逐步收敛为可维护的编辑器主题 token。
- 精修模板约 4137 行，业务状态、时间线、字幕和渲染逻辑集中在单文件，后续修复状态同步会越来越困难。

## 建议执行顺序

1. 修复并固化 FFmpeg/FFprobe 环境诊断，让完整自检通过。
2. 统一字幕编辑状态源，修复退格、撤销、右栏同步和保存语义。
3. 加任务恢复、重试、取消与僵尸任务回收。
4. 给 AI 断句建立“不丢字、不重字、顺序不变”的硬校验和真实视频基准集。
5. 处理精确预览降级、预设误删和导出版本管理。
6. 新增真正的任务中心和批量标题逐条预览。
7. 再做移动端精简、可访问性与视觉一致性。
8. 公网上线前完成鉴权、隔离、上传限制和密钥安全。
