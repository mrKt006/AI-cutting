# 用户逐字稿格式清单

本项目支持 UTF-8 编码的 `.md` 和 `.txt` 逐字稿。系统使用逐字稿校正 ASR 文本、读取内容标题和封面标题；音频时间仍以 ASR 返回的逐字时间为准。

## 最低要求

一份文件至少需要满足以下条件，才可以正确解析：

- 文件格式是 `.md` 或 `.txt`。
- 文件使用 UTF-8 编码。
- 包含一个明确的“逐字稿”板块。
- 逐字稿板块中有实际口播正文，不能只有 `[待确认]`、`[暂无]` 等占位文字。
- 同一文件中不要重复出现两个同名的逐字稿板块。

内容标题和封面标题不是解析逐字稿的必填项，但建议提供：

- 缺少内容标题时，系统使用 Markdown 一级标题或文件名作为默认值。
- 缺少封面标题时，系统使用内容标题生成封面标题建议。
- 提供标题后，系统可以更准确地生成视频内容标题和封面。

## 推荐的 Markdown 格式

```md
# 文档名称

> 系列：可选
> 产出日期：可选
> 状态：可选

## 内容标题

把成交案例喂给AI，它比你更会找下一个客户

## 封面标题

把成交案例喂给AI
它更会找下一个客户

## 逐字稿

你公司最值钱的数据，可能躺在销售和客户的聊天记录里，没人翻过第二遍。

每一单成交，背后都有一条路径。客户从哪来，被哪句话打动，最后为什么下单。

## 备注

这里可以填写拍摄说明或后期备注。备注不会参与语音匹配。
```

Markdown 解析规则：

- `## 内容标题`：视频播放期间显示的标题。
- `## 封面标题`：导出封面使用的标题。
- `## 逐字稿`：参与 ASR 对齐、纠错、断句和自动剪辑的口播正文。
- 其他二级标题板块：默认忽略，不参与语音匹配。
- 一级标题 `# ...`：作为文档名称，并可在缺少内容标题时作为回退值。

## 推荐的 TXT 格式

```text
【内容标题】
把成交案例喂给AI，它比你更会找下一个客户

【封面标题】
把成交案例喂给AI
它更会找下一个客户

【逐字稿】
你公司最值钱的数据，可能躺在销售和客户的聊天记录里，没人翻过第二遍。

每一单成交，背后都有一条路径。客户从哪来，被哪句话打动，最后为什么下单。

【备注】
这里可以填写拍摄说明或后期备注。
```

TXT 解析规则与 Markdown 相同，只是使用中文方括号标记板块。

## 兼容现有 Markdown 文档

对于已经使用以下板块的文档，系统按这个优先级选择逐字稿：

1. `## 逐字稿` 中的有效正文。
2. `## 最终文案` 中的有效正文。
3. `## 原始文案` 中的有效正文。

如果 `最终文案` 是 `[待确认]`、`[暂无]` 或空内容，系统继续使用 `原始文案`，不会把占位文字当作口播。

## 换行规则

- 内容标题和封面标题中的手动换行属于强制换行，最终排版必须保留。
- 标题没有手动换行时，系统根据画幅、预设、真实文字宽度和语义自动换行。
- 逐字稿中的普通换行只表示段落，不直接等于字幕断句。
- 逐字稿可以使用 `|` 标记人工建议断点；`|` 不会显示在最终字幕中。
- 系统自动断句不能漏字、多字、改变原文顺序或拆开固定词组。

示例：

```text
【逐字稿】
你买再好的工具|AI读不到你们业务的上下文，也无法替你做出正确判断。
```

## 额外内容

额外内容不会让整份文档解析失败，但必须放在独立板块中：

- `备注`、`修改记录`、`拍摄说明`、`参考资料`等未知板块会被忽略。
- 板块外的零散文字会被忽略并产生提示。
- 写在逐字稿板块下面的备注会被视为口播正文，因此不要把后期说明混在逐字稿里。
- 重复的内容标题、封面标题或逐字稿板块应当提示用户检查，不应由系统静默选择。

## 多视频任务

- 每条视频分别上传一份逐字稿文件。
- 每份逐字稿只描述对应的视频，不在一个文件中混写多条视频。
- 页面应明确显示视频文件与逐字稿文件的对应关系，不能只依赖上传顺序猜测。

## 提交前检查

- [ ] 文件后缀是 `.md` 或 `.txt`。
- [ ] 文件编码是 UTF-8。
- [ ] 有且只有一个有效的逐字稿来源。
- [ ] 逐字稿正文不是占位文字。
- [ ] 内容标题与封面标题需要的人工换行已经写好。
- [ ] 后期说明没有写进逐字稿正文。
- [ ] 专有名词和英文大小写正确，例如 `AI获客系统`。
- [ ] 多视频任务中的每个视频都有对应的逐字稿文件。
