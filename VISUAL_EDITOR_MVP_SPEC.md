# AI-cutting 口播精修编辑器 MVP Spec

## 1. Summary

本 Spec 定义 AI-cutting 的第一版可视化精修编辑器。

目标不是复刻完整剪映，而是先验证一个关键闭环：

```text
自动粗剪 + ASR 字幕
-> 用户看到逐句稿和时间轴
-> 用户手动删句子 / 改字幕 / 改标题
-> 保存编辑计划
-> 重新导出成片
```

第一版只做“句子级精修”，不做完整多轨剪辑软件。只要这个闭环可用，后续再扩展到完整视频轨、字幕轨、标题轨、拖拽、拆分、合并、AI 建议等能力。

## 2. Product Goal

AI-cutting 当前已经能自动生成成片，但自动剪辑必然会有问题：

- 有些废话没删掉
- 有些句子用户想手动删
- 有些字幕识别错
- 有些断句不自然
- 有些标题没有按用户想法显示

本功能要解决的是：

```text
让用户不用打开剪映，也能完成最基础的人工精修。
```

第一版成功标准：

用户能在浏览器里看到句子，点击播放，手动删除某句，修改字幕文本，修改标题，然后重新导出视频。

## 3. User Flow

### 3.1 推荐流程

```text
首页上传视频
-> 系统自动处理
-> 任务完成页
-> 点击“进入精修”
-> 在精修页修改句子/字幕/标题
-> 保存
-> 重新导出
-> 下载精修版视频
```

### 3.2 为什么第一版放在任务完成后

不建议一开始做“处理前进入编辑器”，原因：

- 当前系统已经有稳定的自动处理链路。
- 精修编辑器需要依赖 ASR 分句结果和剪辑后时间轴。
- 任务完成后再编辑，数据更完整，风险更低。
- 先验证人工精修是否有价值，再决定是否改主流程。

## 4. Scope

### 4.1 MVP 要做

- 新增精修页：

```text
GET /jobs/{job_id}/edit
```

- 从任务输出读取：
  - 火山 ASR 分句
  - 当前字幕文本
  - 当前剪辑报告
  - 输出视频路径
  - 当前标题

- 自动生成或读取：

```text
edit_project.json
```

- 精修页展示：
  - 视频预览
  - 逐句字幕列表
  - 简化时间轴
  - 当前句子属性面板
  - 标题编辑区

- 用户可以：
  - 点击句子定位视频
  - 修改字幕文本
  - 删除句子对应的视频片段
  - 恢复已删除句子
  - 修改封面标题/视频标题文本
  - 保存编辑计划
  - 重新导出精修版视频

### 4.2 MVP 不做

- 不做完整剪映式自由时间线
- 不做拖拽调整视频片段
- 不做拖拽字幕块
- 不做单句字幕起止时间微调
- 不做逐字字幕
- 不做多视频混剪
- 不做素材库
- 不做转场
- 不做贴纸、音效、B-roll
- 不做关键帧
- 不做蒙版
- 不做复杂花字
- 不做 LLM 自动删除内容

## 5. Page Design

### 5.1 页面结构

第一版布局：

```text
顶部工具栏
├── 返回任务
├── 保存
├── 撤销 MVP 可不做
├── 重新导出
└── 状态：未保存 / 已保存 / 导出中 / 导出完成

主体
├── 左侧：句子列表
├── 中间：视频预览 + 简化时间轴
└── 右侧：属性面板
```

### 5.2 左侧：句子列表

每条句子显示：

```text
00:01.20 - 00:03.80
这一年跑下来，我最大的感受是效率真的变了。
```

每条句子操作：

- 点击：视频跳到该句开始时间
- 编辑：修改字幕文本
- 删除视频：删除这一句对应视频和字幕
- 恢复：恢复已删除句子

状态：

- 正常
- 已修改
- 已删除

视觉建议：

- 正常句子：普通显示
- 已修改：显示小标签“已改”
- 已删除：灰色、划线或半透明
- 当前播放句：高亮

### 5.3 中间：视频预览

需要有：

- HTML video 播放器
- 当前字幕叠加预览
- 当前标题叠加预览
- 播放/暂停
- 当前时间
- 点击句子后自动 seek

MVP 可以先播放现有成片，不强制实时跳过被删除片段。

删除后的最终效果以重新导出结果为准。

### 5.4 中间底部：简化时间轴

MVP 时间轴不做自由拖拽，只做可视化参考。

显示：

```text
时间刻度
视频保留/删除区域
字幕句子块
```

交互：

- 点击句子块定位到视频
- 已删除句子块显示灰色
- 当前句子块高亮

不做：

- 拖拽
- 拉伸
- 多轨重排

### 5.5 右侧：属性面板

选中句子后显示：

- 字幕文本 textarea
- 开始时间
- 结束时间
- 状态：正常/已删除/已修改
- 删除视频按钮
- 恢复按钮

标题区显示：

- 封面标题 textarea
- 视频内标题 textarea
- 是否导出标题到视频
- 标题样式预设 MVP 可只显示当前值，不做完整样式编辑

## 6. Data Model

### 6.1 新增文件

每个任务目录新增：

```text
jobs/{job_id}/edit_project.json
```

### 6.2 edit_project.json 示例

```json
{
  "version": 1,
  "job_id": "20260709-120000-ab12cd",
  "source_video": "jobs/.../input/001/video.mp4",
  "current_video": "jobs/.../output/001/xxx.mp4",
  "duration": 120.52,
  "style_preset_id": "default-white",
  "title": {
    "cover_text": "这一年，我重新理解了 AI 工具",
    "video_text": "",
    "show_video_title": false
  },
  "sentences": [
    {
      "id": "s001",
      "start": 0.42,
      "end": 2.8,
      "text": "这一年跑下来",
      "original_text": "这一年跑下来",
      "enabled": true,
      "remove_video": false,
      "edited": false
    }
  ],
  "remove_segments": [],
  "updated_at": "2026-07-09T12:00:00+08:00"
}
```

### 6.3 句子字段说明

```text
id             句子 ID
start          当前视频时间线开始时间
end            当前视频时间线结束时间
text           当前字幕文本
original_text  原始 ASR 字幕文本
enabled        字幕是否启用
remove_video   是否删除对应视频片段
edited         用户是否改过文本
```

### 6.4 删除片段字段

```json
{
  "id": "r001",
  "start": 12.3,
  "end": 14.1,
  "reason": "user_deleted_sentence",
  "sentence_id": "s008"
}
```

## 7. Backend API

### 7.1 打开精修页

```text
GET /jobs/{job_id}/edit
```

逻辑：

- 如果 `edit_project.json` 存在，直接读取。
- 如果不存在，从任务输出初始化。
- 返回编辑器页面。

### 7.2 获取编辑项目 JSON

```text
GET /api/jobs/{job_id}/edit-project
```

返回 `edit_project.json`。

### 7.3 保存编辑项目

```text
POST /api/jobs/{job_id}/edit-project
```

Body：

```json
{
  "title": {},
  "sentences": [],
  "remove_segments": []
}
```

保存到：

```text
jobs/{job_id}/edit_project.json
```

### 7.4 重新导出

```text
POST /jobs/{job_id}/render-edited
```

逻辑：

- 读取 `edit_project.json`
- 根据 `remove_video=true` 的句子生成删除区间
- 重新生成字幕
- 重新烧录字幕
- 重新生成封面
- 输出到：

```text
jobs/{job_id}/edited/
  精修版.mp4
  精修版-封面.jpg
  subtitle.ass
  edit_project.json
```

MVP 可以同步执行，也可以后台线程执行。建议使用后台线程，避免长请求卡住。

## 8. Render Pipeline

### 8.1 MVP 渲染策略

为了降低复杂度，MVP 可以基于“当前成片”二次处理：

```text
当前成片 final.mp4
-> 删除用户标记的片段
-> 重新生成字幕 ASS
-> 重新烧录字幕
-> 输出 精修版.mp4
```

优点：

- 实现快
- 不需要重新跑火山 ASR
- 不影响原任务输出

缺点：

- 如果原成片已经烧录字幕，二次烧录可能出现旧字幕残留

因此更推荐：

```text
剪后无字幕视频 working_video
-> 应用用户删除片段
-> 重新生成 ASS
-> 烧录字幕
```

但当前系统未必保留无字幕中间视频。

### 8.2 MVP 折中方案

第一版可以先：

- 要求任务勾选导出字幕和报告。
- 或在任务处理时额外保留一个无字幕剪后视频到 `work/`。
- 如果没有无字幕视频，则提示“当前任务不支持精修导出，请重新处理并启用精修素材保留”。

更好的改法：

新增内部保留：

```text
jobs/{job_id}/work/{item_id}/cut_no_subtitles.mp4
```

不显示给普通用户下载，但供精修导出使用。

### 8.3 后续优化

成熟后应该从原始视频 + edit plan 重新渲染，而不是基于成片二次剪。

最终管线：

```text
source_video
-> auto_cut keep segments
-> user remove segments
-> final keep segments
-> cut clean video
-> generate edited subtitles
-> burn subtitles
-> generate cover
```

## 9. Frontend State

MVP 前端状态：

```js
{
  selectedSentenceId: "s001",
  dirty: false,
  saving: false,
  rendering: false,
  project: {}
}
```

必须支持：

- 选择句子
- 修改文本
- 删除/恢复
- 保存
- 离开页面前如果未保存，提示用户

可选：

- Undo/Redo 暂不做
- 快捷键暂不做

## 10. Implementation Plan

### Step 1：初始化 edit_project.json

- 从 `volcengine_segments.json` 或字幕文件初始化句子。
- 如果没有 ASR JSON，则从 `subtitle.srt` 解析。
- 如果都没有，则精修页提示无法初始化。

### Step 2：实现精修页静态布局

- 左侧句子列表
- 中间视频
- 底部简化时间轴
- 右侧属性面板

### Step 3：实现句子交互

- 点击句子 seek 视频
- 修改文本
- 删除/恢复
- 保存 JSON

### Step 4：实现重新导出

- 读取 edit project
- 生成删除片段
- 生成新字幕
- 调 FFmpeg 输出精修版

### Step 5：任务页接入口

任务完成页新增按钮：

```text
进入精修
```

如果缺少精修所需素材，则显示：

```text
当前任务缺少精修素材，请重新处理。
```

## 11. Test Plan

### 11.1 初始化测试

- 有 `volcengine_segments.json` 的任务能进入精修。
- 没有 ASR JSON 但有 SRT 的任务能降级初始化。
- 什么都没有时显示清楚错误。

### 11.2 编辑测试

- 点击句子后视频跳转。
- 修改字幕后列表和右侧同步更新。
- 删除句子后时间轴显示已删除。
- 恢复句子后状态正常。
- 保存后刷新页面修改仍存在。

### 11.3 导出测试

- 删除一句后重新导出，视频中该句被删。
- 修改字幕后重新导出，成片使用新字幕。
- 标题修改后封面使用新标题。
- 原始输出不被覆盖。

### 11.4 回归测试

- 不进入精修时，原有任务流程不受影响。
- 批量任务至少能选择其中一个视频精修。
- Web 日志不泄露火山 Token。

## 12. Risks

### 12.1 最大风险：二次烧录字幕

如果基于已经烧字幕的 final.mp4 重新烧录，会出现旧字幕残留。

解决：

- 尽快保留无字幕剪后视频。
- 精修导出优先使用无字幕素材。

### 12.2 时间轴映射复杂

自动剪辑后，原始时间和成片时间不同。

MVP 先基于成片时间线编辑，后续再做原始时间线映射。

### 12.3 删除句子误差

ASR 句子边界不一定精确。

MVP 先允许句子级删除，后续再加前后微调。

### 12.4 批量任务复杂

批量任务有多个视频和多个输出目录。

MVP 可以先支持单个 item 精修：

```text
/jobs/{job_id}/edit?item=001
```

## 13. Done Definition

MVP 完成标准：

- 任务页有“进入精修”入口。
- 精修页能显示视频、句子、时间轴。
- 用户能改字幕。
- 用户能删除/恢复句子。
- 用户能改标题。
- 用户能保存编辑项目。
- 用户能重新导出精修版。
- 原始任务输出不被覆盖。
- 单视频完整跑通。

## 14. Final Recommendation

第一版不要叫“剪辑软件”。

产品里可以叫：

```text
精修
```

技术上叫：

```text
口播精修编辑器 MVP
```

核心价值：

```text
自动剪完以后，让用户快速人工修掉不满意的句子和字幕。
```
