# AI-cutting 完整可视化剪辑器 Roadmap Spec

## 1. Summary

本 Spec 描述 AI-cutting 从“口播精修 MVP”逐步升级为“轻量剪辑软件”的完整路线。

目标体验参考剪映/CapCut 的核心工作流：

```text
视频预览
视频轨道
字幕轨道
标题轨道
属性面板
可视化删减
字幕直接修改
最终导出
```

但产品定位不是通用剪辑软件，而是：

```text
面向口播视频的 AI 自动粗剪 + 人工可视化精修工具。
```

## 2. First Principles

完整剪辑器的本质不是“界面像剪映”，而是稳定处理以下对象。

### 2.1 时间

所有内容必须落在时间轴上：

- 视频片段
- 音频片段
- 字幕片段
- 标题片段
- 删除区间
- AI 建议区间

### 2.2 轨道

轨道用于分层管理不同类型内容。

推荐轨道：

```text
video_track
audio_track
subtitle_track
title_track
ai_suggestion_track
```

MVP 只做简化时间轴，完整阶段再做真正轨道系统。

### 2.3 片段

片段是用户操作的最小单位。

视频片段：

- 拆分
- 删除
- 恢复
- 移动
- 调整边界

字幕片段：

- 修改文本
- 拆分
- 合并
- 调整时间
- 应用样式

标题片段：

- 修改文本
- 调整位置
- 设置显示时间
- 应用标题样式

### 2.4 预览

预览必须尽量接近最终导出。

优先级：

1. 当前画面正确
2. 当前字幕正确
3. 当前标题正确
4. 删除片段后播放逻辑正确
5. 样式接近最终渲染

### 2.5 导出

所有编辑行为最终都要转成一个渲染计划。

最终导出不应该依赖前端状态，而应该依赖保存后的项目文件：

```text
edit_project.json
edit_plan.json
subtitle.ass
render_manifest.json
```

## 3. Product Position

### 3.1 产品不是

- 不是完整剪映复刻
- 不是 Premiere
- 不是 DaVinci Resolve
- 不是通用多轨素材剪辑软件
- 不是特效软件

### 3.2 产品是

- 自动剪辑后的精修工具
- 口播视频字幕修正工具
- 句子级删除工具
- 标题和封面调整工具
- 后续 AI 建议审核工具

### 3.3 核心优势

```text
自动粗剪已经帮用户完成 70%
可视化精修让用户完成最后 30%
```

这比从零做完整剪辑软件更符合当前项目价值。

## 4. Architecture Direction

### 4.1 不建议直接整合完整开源剪辑器

原因：

- 架构和当前 Python/FFmpeg 后端不一致。
- 项目会迅速复杂化。
- 很多能力用不上。
- 开源项目自身稳定性不一定足够。

### 4.2 可借鉴的开源方向

可以研究：

- OpenCut：开源 CapCut 替代方向，适合看产品结构。
- OpenReel Video：浏览器视频编辑器方向，适合看 WebCodecs/WebGPU 思路。
- Twick：React 视频编辑 SDK，适合研究时间线和导出模型。
- ffmpeg.wasm：适合纯前端小文件实验，但大视频性能风险高。
- Remotion：适合代码化视频渲染，不适合直接做剪辑器 UI。

### 4.3 推荐架构

继续保留现有后端：

```text
Python + FastAPI + FFmpeg + ASS + 火山 ASR
```

新增前端编辑层：

```text
HTML/CSS/JS MVP
后续可迁移到 React/Svelte
```

新增项目数据层：

```text
edit_project.json
edit_plan.json
render_manifest.json
```

## 6. Stage Plan

## Stage 1：口播精修 MVP

对应：

```text
VISUAL_EDITOR_MVP_SPEC.md
```

目标：

- 能看句子
- 能改字幕
- 能删句子
- 能改标题
- 能重新导出

不追求完整轨道交互。

## Stage 2：字幕轨道编辑

新增：

- 字幕块轨道
- 双击字幕块编辑
- 拆分字幕
- 合并字幕
- 单句时间微调
- 全局字幕偏移可视化

验收：

- 用户能修复断句问题。
- 用户能修复字幕跟不上或提前的问题。
- 字幕编辑后导出结果正确。

## Stage 3：视频轨道编辑

新增：

- 真正的视频 clip 轨道
- 删除 clip
- 拆分 clip
- 调整边界
- 播放头拖动
- 时间轴缩放

验收：

- 用户能像剪辑软件一样删减片段。
- 删除片段后字幕自动跟随。

## Stage 4：标题轨道

新增：

- 标题 clip
- 多标题
- 标题时间段
- 标题位置
- 标题样式
- 封面标题和视频内标题分离

验收：

- 用户能创建顶部标题。
- 用户能控制标题出现和消失。
- 封面标题不再只依赖首页输入。

## Stage 5：音频波形和精确剪辑

新增：

- 音频波形
- 静音区域显示
- 长停顿标记
- 拖动边界时吸附波形/字幕

验收：

- 用户能根据波形精确删气口。
- 删除点更自然。

## Stage 6：AI 建议审核

新增：

- `edit_suggestions.json`
- AI 建议轨
- 用户确认/拒绝
- 批量应用

验收：

- AI 可以标出疑似废话、重复、错别字。
- 用户确认后才执行。
- 不自动误删重要内容。

## Stage 7：完整产品化

新增：

- 自动保存
- 版本历史
- 更好的导出队列
- 多用户隔离
- 云端部署安全
- 项目管理

## 7. Data Model Roadmap

### 7.1 edit_project.json

完整项目状态。

```json
{
  "version": 2,
  "tracks": {
    "video": [],
    "audio": [],
    "subtitles": [],
    "titles": [],
    "suggestions": []
  },
  "styles": {},
  "metadata": {}
}
```

### 7.2 edit_plan.json

给渲染器使用的简化计划。

```json
{
  "keep_segments": [],
  "remove_segments": [],
  "subtitle_cues": [],
  "title_cues": [],
  "render_settings": {}
}
```

### 7.3 render_manifest.json

记录导出结果。

```json
{
  "input": {},
  "outputs": {},
  "duration": {},
  "created_at": "",
  "errors": []
}
```

## 8. Rendering Roadmap

### 8.1 MVP 渲染

基于当前剪辑结果重新导出。

### 8.2 稳定版渲染

从原始视频 + 自动剪辑计划 + 用户编辑计划重新渲染。

### 8.3 高级版渲染

支持：

- 多轨合成
- 标题动画
- 高亮字幕
- 音频波形辅助剪辑
- 批量导出

## 9. Technical Choices

### 9.1 短期

- 保持 FastAPI + Jinja + 原生 JS
- 时间轴用 HTML/CSS div 实现
- 渲染继续用 FFmpeg
- 数据保存 JSON

### 9.2 中期

当交互变复杂后，考虑迁移编辑器页面到：

- React
- Svelte
- Vue

推荐优先考虑 React 或 Svelte。

原因：

- 时间轴状态复杂
- Undo/Redo 复杂
- 多轨交互复杂
- 组件化更好维护

### 9.3 不建议短期使用

- ffmpeg.wasm 处理大视频
- WebGPU/WebCodecs 自己写解码器
- 完整引入大型剪辑器项目

## 10. UX Principles

### 10.1 预览优先

用户调视频，必须先看到结果。

### 10.2 句子优先

口播剪辑的最小语义单位是句子，不是帧。

### 10.3 删除可恢复

任何删除都不能不可逆。

### 10.4 原始输出不覆盖

精修版必须另存，不能覆盖自动生成的原始成片。

### 10.5 AI 只建议，不擅自剪

除非用户明确开启自动应用，否则 AI 不直接删除视频。

## 11. Key Risks

### 11.1 复杂度失控

如果一开始就追求剪映全功能，项目会失控。

规避：

- 先做句子级精修。
- 每阶段只验证一个核心能力。

### 11.2 浏览器性能

长视频、多轨、大量字幕块会导致前端卡顿。

规避：

- 虚拟列表
- 时间轴分段渲染
- 限制首版视频长度

### 11.3 渲染时间长

每次改一句都重新导出会慢。

规避：

- 预览和导出分离
- 保存编辑计划
- 用户确认后再导出

### 11.4 时间映射错误

自动剪辑后，原始时间线和成片时间线不同。

规避：

- 明确 source_time 和 timeline_time
- 所有数据结构同时保留两个时间

### 11.5 字幕旧内容残留

基于已烧录视频二次编辑会有旧字幕。

规避：

- 保留无字幕中间视频
- 从无字幕素材重新烧录

## 12. Recommended Next Action

下一步不要直接做完整剪辑器。

建议先实现：

```text
VISUAL_EDITOR_MVP_SPEC.md
```

也就是：

```text
句子列表
视频预览
简化时间轴
字幕修改
句子删除
标题修改
重新导出
```

做完后用 3 条真实口播视频验证：

- 短视频 30 秒
- 正常视频 2-5 分钟
- 压力视频 8-15 分钟

确认这个模型真的好用，再进入 Stage 2。

## 13. Final Product Vision

长期版本的 AI-cutting 应该变成：

```text
上传原视频
AI 自动粗剪
AI 自动生成字幕
AI 标出疑似问题
用户在可视化编辑器里确认和精修
一键导出成片和封面
```

这不是完整剪映替代品，而是更适合口播创作者的轻量精修工具。
